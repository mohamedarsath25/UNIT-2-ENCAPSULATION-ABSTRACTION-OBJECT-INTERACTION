Unit II focuses on the core principles of Object-Oriented Programming (OOP) in Java, particularly Encapsulation, Abstraction, Interfaces, Composition, and Object Relationships.

This unit helps students understand how to design secure, modular, and reusable software systems.

**1. Encapsulation**

Encapsulation is the process of wrapping data and methods into a single unit (class) and restricting direct access using access modifiers.

Topics covered:

Need for Encapsulation

Data protection

Getters and Setters

Validation rules

Access Modifiers (private, public, protected, default)

Input security and controlled access

Students learn how to:

Protect sensitive data

Implement validation logic

Create secure class structures
**2. Abstraction**

Abstraction is the process of hiding implementation details and showing only essential features.

Topics covered:

Abstraction fundamentals

Abstract classes

Abstract methods

Partial abstraction

Constructor behavior in abstract classes

Students learn how to:

Hide complexity

Design base classes

Create reusable abstract structures

Use abstraction in real-world examples (ATM, Shape, Employee, etc.)

**3. Interfaces**

Interfaces provide 100% abstraction and enable contract-based design.

Topics covered:

Interface concept

Implementing interfaces

Multiple inheritance using interfaces

Contract-based programming

Students learn how to:

Define behavior without implementation

Achieve multiple inheritance

Build loosely coupled systems

**4. Composition and Object Relationships**

Composition models real-world relationships between objects.

Topics covered:

HAS-A relationship

IS-A vs HAS-A

Lifecycle dependency

Modeling real-world objects

Composition vs Inheritance

Students learn how to:

Design better object relationships

Avoid incorrect inheritance

Build layered object architecture
